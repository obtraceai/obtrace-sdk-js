Obtrace SDK JS

Purpose:
- Browser and Node/Bun SDK for telemetry, replay, context propagation, and ingest transport.

Key entrypoints:
- @obtrace/sdk-js/browser -> initBrowserSDK
- @obtrace/sdk-js/node -> initNodeSDK

Ingest endpoints:
- POST /otlp/v1/logs
- POST /otlp/v1/traces
- POST /otlp/v1/metrics
- POST /ingest/replay/chunk
- POST /ingest/replay/recipes

Design constraints:
- SDK is thin/dumb.
- Business logic is server-side (ingest/control-plane/backend).

Core capabilities:
- Browser: errors, vitals, fetch instrumentation, replay by DOM reconstruction.
- Node/Bun: logs/metrics/spans, fetch instrumentation, unhandled error hooks.
- Wrappers: React, Next, Vue, Angular, Svelte, Express, Fastify, Hono, Nest, Next route handlers.
