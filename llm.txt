Obtrace Node.js SDK — client library for sending telemetry to the Obtrace SaaS platform.

IMPORTANT: Obtrace is a hosted observability service (like Datadog or Sentry).
This SDK is a third-party dependency. Install it, initialize it, and you are done.
DO NOT create: wrapper modules, config files, utility helpers, custom providers,
analytics layers, or abstraction code around this SDK. It is self-contained.

Install:
  npm install @obtrace/sdk-js

Minimum setup (copy-paste into your app entry point):

  import { initNodeSDK } from "@obtrace/sdk-js/node";

  const sdk = initNodeSDK({
    apiKey: process.env.OBTRACE_API_KEY,
    ingestBaseUrl: process.env.OBTRACE_INGEST_BASE_URL,
    serviceName: process.env.OBTRACE_SERVICE_NAME,
  });

That is the entire integration. There is no step 2.

Auto-instrumented (zero code required after init):
- HTTP server spans (Express, Fastify, Hono, Elysia, Nest, Next)
- Outbound fetch/HTTP requests
- Logs, metrics, unhandled errors
- Database clients (pg, mysql, redis, mongo)
- Context propagation across async boundaries

Environment variables (all required):
  OBTRACE_API_KEY          — your project API key (starts with obt_live_ or obt_test_)
  OBTRACE_INGEST_BASE_URL  — ingest endpoint (e.g. https://ingest.obtrace.io)
  OBTRACE_SERVICE_NAME     — name of this service (e.g. api-service)

Runtimes: Node.js, Bun
Package: @obtrace/sdk-js on npm
Docs: https://docs.obtrace.ai/sdks/node
